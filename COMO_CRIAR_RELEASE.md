# 📦 Como Criar uma Release no GitHub

## Passo a Passo Simples

### 1️⃣ Acesse seu Repositório
Vá para: https://github.com/douglas19961/appatualizarteste

### 2️⃣ Vá para a Página de Releases
- Clique na aba **"Releases"** no lado direito da página
- OU acesse diretamente: https://github.com/douglas19961/appatualizarteste/releases

### 3️⃣ Criar Nova Release
- Clique no botão **"Create a new release"** (ou "Draft a new release")

### 4️⃣ Preencher os Campos

#### Tag de Versão
- Escolha uma tag como: `v1.0.0` ou `1.0.0`
- Se for a primeira vez, você pode criar uma nova tag
- Exemplos de tags: `v1.0.0`, `v1.0.1`, `v2.0.0`

#### Título da Release
- Exemplo: "Versão 1.0.0 - Primeira Release"
- Ou simplesmente: "v1.0.0"

#### Descrição (Opcional)
- Você pode descrever o que há de novo nesta versão
- Exemplo:
  ```
  Primeira versão do sistema de atualização
  
  Funcionalidades:
  - Verificação de atualizações
  - Download automático
  ```

### 5️⃣ Anexar Arquivos (Opcional)
- Você pode arrastar arquivos ou clicar em "Attach binaries"
- Exemplos: executáveis (.exe), instaladores, arquivos ZIP
- Isso permite que os usuários baixem os arquivos diretamente

### 6️⃣ Publicar
- Clique em **"Publish release"** (ou "Publish" se for draft)
- Pronto! A release está publicada

## ✅ Verificar se Funcionou

Depois de criar a release, você pode:
1. Voltar para a aplicação Delphi
2. Clicar em "Verificar Atualizações"
3. A release deve aparecer!

## 💡 Dicas

- **Tags**: Use sempre o mesmo formato (ex: sempre com "v" ou sempre sem)
- **Versões**: Siga o padrão semântico: MAIOR.MENOR.PATCH (ex: 1.0.0, 1.0.1, 1.1.0, 2.0.0)
- **Releases**: Você pode criar quantas releases quiser, cada uma com uma versão diferente
